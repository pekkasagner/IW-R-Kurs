Grundlagen der Programmierung in R

Hier werden ausgewählte Grundlagen der Programmierung in R kurz erklärt. Auch wenn in dieser Einführung keine Vorkenntnisse in R vorausgesetzt werden, empfiehlt es sich, sich mit den Fundamenten der Programmiersprache R vor Beginn des Kurses auseinanderzusetzten. So bekommt man ein Gefühl dafür, wie R funktioniert und kann in vielen Situationen flexibler auf eventuelle Probleme und eigene Anpassungswünsche reagieren.

Coding Grundlagen

R kann wie ein gewöhnlicher Taschenrechner benutzt werden.

1 / 10 * 90
## [1] 9
sin(pi / 2)
## [1] 1

Neue Objekte können mit dem “assignment operator” <- erzeugt werden. Alternativ kann für das Assignment auch = verwendent. Aus konventionellen Gründen der besseren Lesbarkeit wird in R für die Zuweisung von Objekten aber der <- Operator verwendent.

a <- 5
b <- 6
a * b
## [1] 30

Objektnamen

Objektnamen müssen in R mit einem Buchstaben beginnen und können nur Buchstaben, _ oder . enthalten.

beispiel_name <- "Hans"
anderes_beispiel <- "Peter"

Funktionen

In R gibt es eine große Anzahl an Funktionen, die in Base-R enthalten sind und direkt genutzt werden können. Ein typischer Aufrauf einer Funktion sieht folgendermaßen aus:

function_name(arg1 = val1, arg2 = val2, ...)

Eine solche Funktion ist etwa mean(), die das arithmetische Mittel der eingegebenen Werte ausgibt oder seq(), die eine Sequenz aus einem Start- und Endpunkt erzeugen kann.

mean(c(5,34,57))
## [1] 32
seq(1,10, length.out = 5)
## [1]  1.00  3.25  5.50  7.75 10.00

Setzt man dem Namen der jeweiligen Funktion ein ? voran, kann die Hilfe-Seite der Funktion direkt in R-Studio angezeigt werden.

?seq
## starting httpd help server ... done

Vektoren

In R gibt es zwei Typen von Vektoren

  1. Atomic Vektoren mit den Typen logical, integer, double, charachter, complex und raw. Integer und double werden dabei häufig kollektiv als numeric bezeichnet.

  2. Listen, die manchmal auch als rekursive Vektoren bezeichnet werden, da Listen weitere Listen beinhalten können.

Der Unterschied zwischen atomic Vektoren und Listen ist der, dass atomic Vektoren immer homogen sein müssen, also nur Objekte des gleichen Typs enthalten können, während Listen heterogen sind.

Typen von Vektoren

  1. Logical

Können nur drei mögliche Werte annehmen: TRUE, FALSE oder NA

1:10 %% 3 == 0
##  [1] FALSE FALSE  TRUE FALSE FALSE  TRUE FALSE FALSE  TRUE FALSE
c(TRUE, TRUE, FALSE, NA)
## [1]  TRUE  TRUE FALSE    NA
  1. Numeric

Integer und double Vektoren werden in R unter numeric gefasst. Zahlen sind in R standardmäßig doubles, um einen integer Vektor zu erstellen muss hinter die Zahl ein L angehängt werden.

typeof(1)
## [1] "double"
typeof(1L)
## [1] "integer"
  1. Character

Character Vektoren können eine beliebige Anzahl sogenannter Strings ( eine zusammenhängende Folge von Zeichen) enthalten.

x <- "Das ist ein String"
x
## [1] "Das ist ein String"
